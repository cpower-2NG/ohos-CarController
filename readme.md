# CarControler

开源鸿蒙手机端平衡车控制应用，面向实验八场景开发。当前仓库包含鸿蒙前端应用与配套嵌入式控制代码，适合用于网络协同开发和后续联调。

## 项目简介

本项目目标是完成一个可用于平衡车控制的鸿蒙手机应用，逐步实现车辆控制、AI 对话、语音识别、导航管理和车辆管理等能力。

## 项目框架

本仓库当前采用“前端应用 + 嵌入式控制”两层结构：

- 前端应用：负责界面展示、TCP 连接、指令发送、状态显示
- 嵌入式控制：负责接收控制指令、执行硬件动作、协议转发

### 目录结构

- `CarControler/`：鸿蒙手机前端工程
	- `AppScope/`：应用级配置
	- `entry/`：主应用模块
	- `entry/src/main/ets/pages/Index.ets`：主页面与 Tab 框架
	- `entry/src/main/ets/models/TcpClient.ets`：TCP 客户端封装
- `car_control_embedded/`：嵌入式控制代码
	- `uart_tcp_re/`：TCP 服务端 + 平衡车控制逻辑
	- `uart_tcp_sta/`：另一套车控样例

## 五大功能

### 1. 实现车辆控制

- 连接平衡车服务端
- 发送前进、后退、左转、右转、停止指令
- 支持按住控制、松开发停止
- 显示连接状态、回包和最近指令

### 2. 实现 AI 智能对话

- 输入文本并接入大模型
- 将自然语言解析成控制指令
- 支持扩展到车灯、座椅、氛围灯等控制场景

### 3. 实现 AI 语音识别

- 将语音转换为文字
- 按住语音按钮开始识别，松开后自动发送到 AI
- 识别文本会回填到输入框并进入对话流
- 已支持识别超时与失败重试（最多 3 次）

### 4. 实现导航管理

- 预留地图导航页面
- 支持定位、路线规划与导航展示
- 方便后续接入地图服务能力

### 5. 实现车辆管理

- 管理连接参数
- 预留车辆状态、调试信息和扩展功能入口
- 便于后续扩展设备管理、日志和配置管理

## 开发说明

### 前端工程说明

- 页面入口在 `entry/src/main/ets/pages/Index.ets`
- TCP 逻辑已拆分到 `entry/src/main/ets/models/TcpClient.ets`
- 当前车控页使用 TCP 发送平衡车协议指令
- AI 对话模块位于 `entry/src/main/ets/pages/AiDialogPage.ets`
- AI HTTP 调用与指令解析位于 `entry/src/main/ets/models/AiChatService.ets`
- 语音识别封装位于 `entry/src/main/ets/models/SpeechToTextService.ets`

### 嵌入式工程说明

- `uart_tcp_re/` 中定义了 TCP 命令格式和 UART 转发逻辑
- 车控命令格式已统一为 `$1~$4` 和 `$0` 的协议形式
